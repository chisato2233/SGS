## 3. 架构基线（稳定决策）

### 3.1 服务器权威

- 全部游戏结算只在**服务器**进行，服务器是唯一真相源。客户端不持有可信状态。
- 状态经 UE 复制下发：公共信息（阶段、当前行动者、各家血量/手牌数/装备/标记）走 `GameState`/`PlayerState`；**私密信息（手牌具体内容）只复制给拥有者**（`COND_OwnerOnly` 或定向下发）。
- 玩家指令经**可靠 RPC**上行（`Server` RPC，服务器校验合法性后才执行）。
- 起步用**监听服务器（listen server）**，架构上预留**专用服务器（dedicated）**，不写依赖"本地即服务器"的假设。

### 3.2 决策代理抽象（多人 / AI 并存的关键）

- 逻辑层需要任何玩家输入时，只面向接口 `ISGSDecisionAgent` 发起**决策请求**（如：是否打出闪？弃哪些牌？选择目标？）。
- 真人代理：把请求经 RPC 下发到该玩家 `PlayerController` → UI 收集 → 经 RPC 回传。
- AI 代理：在**服务器侧**直接计算应答（参考 QSanguosha 的价值/威胁评估）。
- 逻辑层**不感知**对端是人还是 AI；因此"全 AI 单机"、"全真人联机"、"混合"三种局面是同一套代码。

### 3.3 异步、非阻塞

- UE 游戏线程不可阻塞等待网络/UI 回复。因此结算流程写成**事件驱动的异步状态机**：请求决策后**挂起**当前流程，待应答回调到达再恢复；**超时**则用默认动作 / AI 托管。
- 这是与 QSanguosha「`RoomThread` 阻塞式 `askForXXX`」的**关键差异**——我们用挂起/恢复（延续/回调）实现等价语义，但不阻塞线程。
- 决策请求需带**唯一 RequestId** 与**期望状态戳**，防止过期/重复应答串话。

### 3.4 事件总线 + 触发器

- 技能 =「触发时机（TriggerEvent）+ 条件 + 效果」，注册到事件系统；结算流程在关键节点广播事件，命中的技能按时机/优先级介入。
- 具体 `ETriggerEvent` 取值集合、优先级与解析顺序在 **Plan 0003** 随事件总线一并定稿（避免现在过度规约）。

### 3.5 数据驱动

- 卡牌/武将/技能的**静态定义**走 `DataTable`/`DataAsset`；运行态实例与效果走 C++（效果类 + 注册表）。
- 新卡优先复用现成效果类；不够用才加新类（YAGNI）。

## 4. 代码分层与文件夹结构

依赖方向（**单向，不得反向**）：`UI → Game → Logic → Core`；`AI → Logic → Core`；`Data → Core`。逻辑层不依赖 UI / 具体网络实现。

```
Source/SGS/
├── SGS.Build.cs
├── SGS.h / SGS.cpp                 # 模块入口
├── Core/                           # 跨层基础设施，谁都可依赖，不依赖任何上层
│   ├── SGSLogChannels.h/.cpp       #   日志分类
│   └── SGSTypes.h                  #   通用枚举/常量（花色/颜色/牌型/阶段/势力）
├── Logic/                          # 服务器权威纯逻辑（核心；不依赖 UI/网络具体实现）
│   ├── Engine/                     #   对局驱动、回合阶段机、事件总线/触发器、结算管线
│   ├── Players/                    #   座位模型 + ISGSDecisionAgent 决策接口
│   ├── Cards/                      #   牌的定义与运行态
│   └── Skills/                     #   技能基类、触发技、效果
├── Game/                           # UE 框架编排层（权威落地点）
│   ├── SGSGameMode (server)        #   持有/驱动 Logic
│   ├── SGSGameState (replicated)   #   公共状态
│   ├── SGSPlayerState (replicated) #   每座位状态（私密信息定向复制）
│   └── SGSPlayerController         #   真人输入/决策 RPC 端点
├── AI/                             # 服务器侧 AI 决策代理（实现 ISGSDecisionAgent，参考 QSanguosha）
├── Data/                           # DataTable 行结构 / DataAsset 定义
└── UI/                             # Native Code-first UI / SGSUI（后期）
```

> 空目录不入库（不放 .gitkeep）；每层在其首个实现 Plan 落地时再建。本计划只建 `Core/`。
