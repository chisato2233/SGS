## 1. 目标

让一个武将技能能够在现有 RuleRegistry 内组合主动规则、响应规则、触发规则、数值修正、视为入口和 AI 评价器，并以咆哮、武圣、奸雄、制衡四个标准技能证明各接缝可实际运行。

## 2. 方案

- 标准武将定义集中声明阵营、体力和技能名，初始化座位时写入权威技能列表。
- 技能包把所有规则部件注册到每局 RuleRegistry，把评价器注册到每局 AI 注册表，不使用全局可变单例。
- Modifier 和 ViewAs 是 RuleRegistry 的查询型入口；它们只修正规则结果或生成候选，不绕过 Command 与服务器复验。
- 主动技能使用独立强类型 Command，并携带服务器候选允许的技能名、素材牌和目标。
- 多牌技能在共享决策结构中表达最小/最大牌数，真人与 AI 使用同一合法候选。

## 3. 真实消费者

- 咆哮：把张飞的出牌阶段杀次数上限修正为不限。
- 武圣：允许关羽把红色手牌作为杀使用，并为后续“打出杀”响应复用同一视为入口。
- 奸雄：受到牌造成的伤害后获得该结算牌。
- 制衡：孙权每回合限一次，弃置任意张牌并摸等量牌。

## 4. 验收

- 四个技能均从武将定义自动生效，不在 GameDriver 按武将名分支。
- 规则层生成合法技能候选，CommandType 与技能规则再次验证素材和次数。
- 技能的移动、摸牌和状态改变继续走 EffectPipeline，触发走 TimingEvent。
- 不新增或运行测试代码；仅做生产代码编译与静态差异检查。
