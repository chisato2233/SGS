## 1. 目标

在保留 SGS 服务器权威、强类型 Command/Rule/Effect、确定性随机与回放审计的前提下，逐步形成接近 NoName 成熟度的 AI 评价、武将技能表达和完整三国杀流程，最终交付可稳定运行的八人标准身份局。

## 2. 约束与取舍

- 真实内容驱动框架，不为尚不存在的状态预建错误协议、通用状态袋、测试夹具或调试日志。
- AI、真人与未来网络输入继续共用 DecisionAgent 和服务器合法候选；AI 不读取未公开身份。
- 技能复用 RuleRegistry、TimingEvent、Modifier/ViewAs 接缝和 EffectPipeline，不建立平行事件引擎。
- 不移植 NoName 的全局状态、DOM 驱动决策、客户端规则执行、裸随机或动态 step 脚本。
- 每个里程碑单独形成可验收子计划；上一阶段拥有真实消费者后再扩展下一阶段。

## 3. 里程碑

### M1 — 信息安全的通用 AI 评价

- [x] AI 仅通过受限 WorldView 观察局面。
- [x] 合法行动统一展开为候选，由全局、卡牌和技能评价器组合评分。
- [x] BasicAI 不再按卡名硬编码决策，稳定平局且保留评分明细。

### M2 — 武将技能运行时

- [ ] 技能定义可组合主动、响应、触发、修正、视为和 AI 评价规则。
- [ ] 用咆哮、武圣、奸雄、制衡分别验证次数修正、视为、伤害后触发和主动多牌选择。
- [ ] 技能状态只随真实技能需求加入权威模型。

### M3 — 完整对局流程

- [ ] 补齐准备、判定、摸牌、出牌、弃牌、结束阶段及其稳定时机。
- [ ] 完成锦囊、装备、延时锦囊、连续响应、濒死、死亡和胜负结算。
- [ ] 取消、替换与修正仍经规则管线并可被回放审计。

### M4 — 标准身份局内容闭环

- [ ] 完成项目标准牌表以及装备、锦囊规则。
- [ ] 首批武将池固定为曹操、司马懿、夏侯惇、刘备、关羽、张飞、孙权、吕布。
- [ ] 支持八人身份分配、选将、完整对局、死亡公开、终局和回放。

### M5 — 身份认知与策略增强

- [ ] AI 根据公开行动维护敌友置信度，不读取权威隐藏身份。
- [ ] 卡牌和技能提供 order/usefulness/effect/threat 等局部语义。
- [ ] 评分稳定后增加有限一层前瞻，不以搜索替代规则语义。

## 4. 工程质量门

- 每项框架能力必须由真实基础牌、技能或阶段消费；只有接口而无消费者不算完成。
- 合法性始终由规则层生成与复验，评价器只能排序候选，不能制造动作。
- 同一公开局面和候选顺序必须得到同一结果；未来需要随机平局时只能接入 RandomAudit。
- 隐藏信息必须在 WorldView 构建处裁剪，不依赖评价器自律。
- 当前阶段不新增自动化测试；每个子里程碑执行 Development Editor 编译和其列明的 PIE 手工场景。
