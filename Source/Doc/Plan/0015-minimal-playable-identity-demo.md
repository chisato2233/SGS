## 1. 目标

实现固定 `1 真人 + 7 AI` 的八人身份局：随机座次与 `1 主公、2 忠臣、4 反贼、1 内奸` 身份，从开局、连续出牌、自动弃牌运行到身份胜负自然结算。匿名角色中主公 5 点体力，其余角色 4 点体力。

## 2. 本轮硬约束

- 不新增测试代码、日志代码、通用框架或当前固定 Demo 不会触发的错误处理；不运行自动化测试。
- 像外科手术一样只实现验收路径需要的最小代码，优先复用 Command/Rule、RandomAudit、EffectPipeline、ActiveEffectTimeline、现有响应 UI 与牌区工具。
- 固定八人；不实现武将、技能、装备、锦囊、判定牌、联网大厅、重连、托管、超时或完整官方牌堆。
- 自动弃牌，不扩展多选协议；身份模式由胜负规则结束，不以 `MaxTurns` 截断。

## 4. 关键规则

- 主公存活且反贼、内奸全部死亡：主公与忠臣胜。
- 主公死亡且唯一存活的非主公为内奸：内奸胜；其他主公死亡情况由反贼胜。
- 杀每个出牌阶段限一次；酒每回合限一次，令下一张杀伤害 `+1` 且无论是否被闪都消费；濒死者本人可用酒回复 1 点体力。
- 响应请求保留单一必需牌兼容字段，并以最小可接受牌名集合统一服务 Command 校验、Snapshot、UI 与 AI。

## 5. 验收标准

- 多次开局时本地真人座次和身份随机变化，身份数量恒定；主公、本人、死亡角色与终局身份按规则可见。
- 同一出牌阶段能连续用牌并以 Pass 结束；弃牌阶段能稳定将手牌自动弃至体力上限。
- 杀、闪、桃、酒、酒杀 2 点伤害、濒死自救以及杀/酒次数限制均能在真实对局中工作。
- 反贼击杀奖励、主公误杀忠臣惩罚和三类身份终局判定正确。
- AI 不攻击同阵营，能使用基础牌推进对局；一名真人加七名 AI 无需外部改状态即可自然结束整局。
- 终局持续显示本地胜利/失败与获胜身份，不新增专用结算 Widget。
